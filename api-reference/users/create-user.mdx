---
title: "Create user"
description: "Create a new user account. Requires an active session with sufficient role permissions."
---

Creates a new user account. The action is role-gated server-side: only `Gefe`, `Jefe`, and `Supervisor` roles can create users, and each can only create users of roles lower than their own.

**Endpoint:** `USERS` service — `https://script.google.com/macros/s/AKfycbxAqyEcAHetH6yN4qccGILL-L3IzMSPVuVJ1kpuO86GqfDXTKP8cHrrB7UkKN1r_0g5/exec`

**Action:** `createUser`

<Note>
  The server generates the final username automatically if the provided `Nombre_Usuario` conflicts with an existing one — it appends a numeric suffix (e.g., `juan.perez` → `juan.perez1`). Always check the returned message to confirm the actual username created.
</Note>

| Creator role | Roles they can create |
|---|---|
| `Desarrollador` | `Desarrollador`, `Gefe`, `Supervisor`, `Tecnico`, `Tecnico_exterior` |
| `Gefe` / `Jefe` | `Supervisor`, `Tecnico` |
| `Supervisor` | `Tecnico` |

## Request

<ParamField path="action" type="string" required>
  Must be `"createUser"`.
</ParamField>

<ParamField path="payload.sessionToken" type="string" required>
  The active session token of the user performing the creation. Used to verify role permissions.
</ParamField>

<ParamField path="payload.newUser" type="object" required>
  Object containing the new user's details.
  <Expandable title="newUser properties">
    <ParamField path="Nombre_Usuario" type="string" required>
      Desired login username. The server will make it unique if needed by appending a number.
    </ParamField>
    <ParamField path="Password" type="string" required>
      Initial password for the account.
    </ParamField>
    <ParamField path="Privilegios" type="string" required>
      Role to assign. One of `"Tecnico"`, `"Supervisor"`, `"Gefe"`, `"Desarrollador"`, or `"Tecnico_exterior"`. Must be permitted by the creator's own role.
    </ParamField>
    <ParamField path="Nombre_Completo" type="string">
      The user's full display name. Shown in activity logs and as contributor credit on catalog entries.
    </ParamField>
    <ParamField path="Telefono" type="string">
      Phone number (optional).
    </ParamField>
    <ParamField path="Correo_Electronico" type="string">
      Email address (optional).
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="status" type="string" required>
  `"success"` on success, `"error"` on failure.
</ResponseField>

<ResponseField name="message" type="string">
  On success: `"Usuario '<username>' creado."` where `<username>` is the actual (possibly de-duplicated) username assigned.
</ResponseField>

## Example

```javascript
const USERS_ENDPOINT = 'https://script.google.com/macros/s/AKfycbxAqyEcAHetH6yN4qccGILL-L3IzMSPVuVJ1kpuO86GqfDXTKP8cHrrB7UkKN1r_0g5/exec';

const session = JSON.parse(localStorage.getItem('gpsepedia_session'));

const response = await fetch(USERS_ENDPOINT, {
  method: 'POST',
  headers: { 'Content-Type': 'text/plain' },
  body: JSON.stringify({
    action: 'createUser',
    payload: {
      sessionToken: session.SessionToken,
      newUser: {
        Nombre_Usuario: 'maria.garcia',
        Password: 'initialPass123',
        Privilegios: 'Tecnico',
        Nombre_Completo: 'María García',
        Telefono: '+1-555-0100',
        Correo_Electronico: 'mgarcia@example.com'
      }
    }
  }),
  redirect: 'follow'
});

const result = await response.json();
if (result.status === 'success') {
  console.log(result.message); // "Usuario 'maria.garcia' creado."
}
```
